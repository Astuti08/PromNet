Git

Git adalah sistem kontrol versi populer dan kuat yang digunakan oleh pengembang dan tim pengembangan perangkat lunak untuk mengelola perubahan pada kode sumber proyek perangkat lunak. Sistem ini memungkinkan pengguna untuk melacak semua perubahan yang dilakukan pada file dan folder proyek, mencatat setiap perubahan sebagai entitas terpisah yang disebut "commit," membuat cuplikan proyek pada titik waktu tertentu. Dengan Git, pengembang dapat dengan mudah melacak dan memahami kemajuan proyek, melihat siapa yang melakukan perubahan, apa yang berubah, dan kapan perubahan tersebut terjadi.

Fungsi Utama Git:

  • Kontrol Versi
  • Kolaborasi Tim
  • Manajemen Perubahan
  • Pengujian
  • Pengembangan Paralel
  • Pemulihan
  • Distribusi
  • Pemantauan Perubahan
  • Ketersediaan
  • Branching dan Merging

Perintah Dasar di Git:

  1. Git-Init: untuk membuat repository yang ada pada file lokal dan berlokasi di folder .git.
  2. Git Status: untuk mengecek status repository lokal.
  3. Git add: untuk menambahkan file baru ke repository yang baru dipilih.
  4. Git commit: untuk menyimpan apabila terjadi perubahan dan dilakukan pada repository jarak jauh, namun tidak bisa melakukan perubahan.
  5. Git push: untuk mentransfer perubahan file ke repository jarak jauh setelah melakukan perubahan.
  6. Git Checkout: digunakan untuk menukar cabang aktif dengan cabang yang dipilih.
  7. Git Merge: untuk menggabungkan cabang aktif dengan cabang yang dipilih.
  8. Git Clone: untuk membuat salinan repository lokal.
  9. Git pull: Mengambil perubahan terbaru dari repositori jarak jauh dan menggabungkannya dengan cabang lokal.
  10. Git reset:digunakan untuk mengubah status dan sejarah commit dalam repositori.
